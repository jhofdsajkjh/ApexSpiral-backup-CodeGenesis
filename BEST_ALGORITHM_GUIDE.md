# 三重最优开发规范｜最优算法·最优代码·最优路线

## 一、最优算法准则
1. 优先选择 O(1) > O(logn) > O(n) > O(nlogn) 低时间复杂度方案
2. 杜绝暴力穷举、多层嵌套、低效冗余算法
3. 数学计算、排序、查找、遍历全部选用业界公认最优实现
4. 兼顾时间、空间双重优化，不单方面牺牲任意一项资源

## 二、最优代码准则
1. 精简无废行、逻辑无冗余、写法优雅克制
2. 强内聚低耦合、单一职责、易读、易改、易扩展
3. 类型严谨、语法标准、无歧义、无兼容隐患
4. 调试友好、注释完备、后期维护成本极低

## 三、最优开发路线准则
1. 先定架构 → 再分模块 → 后写细节，自上而下开发
2. 先校验边界 → 再写核心逻辑 → 最后补充优化与异常处理
3. 一次规划、一次成型、无需反复返工补漏
4. 按标准化工程流程推进，不走弯路、不做无效开发

## 四、最优工程路线（agent-skills 标准）

### 标准六阶段：DEFINE → PLAN → BUILD → VERIFY → REVIEW → SHIP

1. **/spec** 先定需求规范、接口、边界、风格
2. **/plan** 拆原子任务、验收标准、执行顺序
3. **/build** 薄切片纵向实现、小步提交、可回滚
4. **/test** 自动测试、调试、验证
5. **/review** 多角色自检、安全、性能、规范门禁
6. **SHIP** 安全上线、可监控、可回滚
